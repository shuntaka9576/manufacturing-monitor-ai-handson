# ch5 完成形: PLaMo Embedding - 意味類似検索

ch5の演習完了後の状態です。ch4までの全機能に加え、PLaMo Embeddingによる意味類似検索ページが追加されています。

## 起動方法

以下の順番で実行してください。

```bash
cd ch5_fin

# 1. 依存パッケージのインストール（初回 or pyproject.toml変更時）
uv sync

# 2. シードデータの生成（SQLiteにデータ投入）
uv run python db/seed.py

# 3. Embeddingの生成（初回実行時にモデル約4GBがダウンロードされます）
uv run python db/embed.py

# 4. アプリの起動
uv run streamlit run app.py
```

## 注意事項

- 手順2→3の順番は必須です。seed.py でステータス変更履歴を生成した後に、embed.py でそのテキストをベクトル化します
- Apple Silicon 環境では bfloat16 演算で NaN が発生するため、モデルを float32 で読み込んでいます
- transformers 5.x では `torch_dtype` が非推奨になりました。`dtype` パラメータを使用してください（`torch_dtype` を使うと NaN が発生します）

## 意味類似検索について

### サンプルクエリと期待結果

以下のクエリを試すと、キーワード一致ではなく意味的類似性で結果が返ることを体験できます。

| クエリ                     | 1位ヒット (類似度)                          | なぜヒットするか                                                                             |
| -------------------------- | ------------------------------------------- | -------------------------------------------------------------------------------------------- |
| 油圧の圧力が下がった       | 油圧圧力が規定値を下回る低下を検知 (74.0%)  | 「下がった」という日常表現を「規定値を下回る低下」という技術的表現に意味レベルで紐付けている |
| 冷却水の温度が高すぎる     | 冷却水温度が上限値を超過 (71.4%)            | 「高すぎる」→「上限値を超過」。口語的な表現と正式な記録文を結びつけている                    |
| 金型交換                   | 新規金型セットアップのため停止 (70.7%)      | 「交換」→「セットアップ」。同じ作業を異なる語彙で表現しても意味的に近いと判定される          |
| 材料不足                   | 材料補充のため一時停止 (65.1%)              | 「不足」→「補充」。不足しているから補充する、という対義語の意味的関係を捉えている            |
| 温度が異常                 | 温度異常上昇を検知 (65.2%)                  | 「異常」という曖昧なクエリから「異常上昇を検知」という具体的な記録にマッチ                   |
| 定期点検                   | 年次法定点検のため計画停止 (64.5%)          | 「定期」→「年次法定」。一般的な表現から、より具体的な制度用語への意味的マッチ                |
| ロボットの位置がずれている | ロボットアーム位置ずれを検知 (64.3%)        | 日常的な言い回し「位置がずれている」を、技術ログ「位置ずれを検知」に対応付けている           |
| 精度が悪い                 | 加工精度低下により調整停止 (59.2%, 3位)     | 「悪い」→「低下」。主観的な表現を定量的な表現に紐付けている                                  |
| 品質に問題がある           | 原材料品質不良によるロット停止 (58.9%, 3位) | 「問題がある」→「不良」。抽象的な問題意識から具体的な品質異常を見つけている                  |

### 現状の課題

PLaMo Embedding による意味類似検索には以下の課題があります。

- **ノイズの多さ**: 「試作品用の設定変更のため停止」「新規金型セットアップのため停止」の2件はほぼ全てのクエリで上位に表示される。短い文書ではベクトルは汎用的になりやすく、どのクエリとも高い類似度を示す傾向がある
- **抽象的なクエリの精度**: 「部品が古くなった」「電気のトラブル」のような抽象度の高いクエリでは、期待する結果が上位にヒットしにくい
- **閾値の調整**: デフォルト閾値 0.55 ではノイズが混じりやすい。具体的なクエリ（「油圧の圧力が下がった」等）では精度が高いが、曖昧なクエリでは精度が低下する

### 改善の方向性

- **文書の前処理**: reason テキストに設備名やカテゴリ情報を付加してからベクトル化することで、文書ベクトルの区別性を向上させる
- **リランキング**: 上位N件を取得した後、LLM でリランキングすることで精度を向上させる
- **ハイブリッド検索**: キーワード検索（BM25等）とベクトル検索を組み合わせることで、キーワード一致と意味的類似の両方を活かす
